# ADR-0002: Model coverage (scGPT only vs scGPT + Geneformer)
Date: 2026-05-19
Status: Proposed — awaiting Armin sign-off

## Context

The probe's results depend on which pretrained single-cell foundation
model(s) it tests. The two most cited are scGPT (Cui et al. 2024,
*Nat Methods*) and Geneformer (Theodoris et al. 2023, *Nature*). They
encode cells differently — scGPT uses a transformer over masked
expression tokens; Geneformer uses a transformer over gene-rank
sequences — so they don't necessarily fail in the same ways on the
same benchmarks. Running both is more informative but doubles the
implementation effort and the README's interpretation burden.

The brief (§6.4 item 1) explicitly asks: "Which model and which
weights?"

## Options considered

### Option A — scGPT only
A single, deeply-probed model. Clean, defensible single conclusion.
- Pros: simpler implementation; one HuggingFace revision to pin; one
  set of failure modes to interpret; less likely to produce
  "well, scGPT failed but Geneformer passed" results that require
  more nuance than the README can comfortably carry.
- Cons: a one-model probe is criticisable — "you cherry-picked
  scGPT because Geneformer would have shown different results."

### Option B — Geneformer only
- Pros: same simplicity benefit; Geneformer's gene-rank framing is
  arguably more interpretable than scGPT's token-level approach.
- Cons: scGPT is more widely cited in 2024-2025 work on PD-relevant
  benchmarks; using only Geneformer skips the model the field
  recognises most.

### Option C — Both, reported side-by-side
Run all three benchmarks on both models. Report per-model results,
plus a brief "where do they agree / disagree" summary.
- Pros: most defensible; convergence between models is a real
  signal; divergence is a real call-out; matches the spirit of
  ADR-0001 (separate ground-truth options reported with explicit
  framing).
- Cons: 2x compute; 2x interpretation surface; the README has to
  resist the temptation to call agreement "validation" (it's not —
  two models trained on overlapping corpora can agree for
  uninteresting reasons).

### Option D — Both, headline one, the other as a robustness check
Designate scGPT as the headline probe; run Geneformer on the same
benchmarks and report its results in a "robustness" section.
- Pros: README clarity (one headline, one robustness check);
  reviewer can still see both numbers.
- Cons: choosing the headline still requires a justification.

## Criteria for choosing
1. **Defendability under hostile review.** A reviewer asking "would
   Geneformer have changed your conclusion?" should get a direct
   numerical answer.
2. **README clarity.** The headline result must be unambiguous;
   readers who only read the first paragraph should not need to
   pick between two model verdicts.
3. **Implementation cost.** Both models have HuggingFace releases
   with documented APIs; the marginal cost of the second is real
   but bounded.
4. **Honest framing on agreement.** ADR-0005 (negative-result
   framing) and §6.4 item 6 both insist on honest reporting; agreement
   between scGPT and Geneformer must not be over-claimed.

## Decision

**Option D: scGPT as the headline probe, Geneformer as the
robustness check.** Headline results in the README's main table use
scGPT; an adjacent "robustness" table reports Geneformer's results
on the same benchmarks. If they disagree substantially (one passes
a benchmark, the other fails), the README's interpretation paragraph
calls it out specifically rather than averaging or hiding the
disagreement.

The scGPT-as-headline choice is defensible because:
1. It is the more widely cited of the two on neurodegeneration tasks
   in current literature.
2. It is the more recent (2024 vs 2023); its training corpus is
   marginally larger.
3. Designating one headline avoids the "which conclusion do I cite?"
   ambiguity for portfolio readers.

## Consequences
- Both models are pinned to specific HuggingFace revision SHAs,
  recorded in `data/README.md`.
- The pretraining-cutoff check (ADR-0001 C1) is performed
  independently for each model — the held-out atlas must postdate
  *both* models' cutoffs.
- The README's "Limitations" section explicitly notes that
  scGPT-headline-Geneformer-robustness was a deliberate
  framing-choice and that the reverse framing would be equally
  defensible.

## Open questions for Armin
1. Confirm Option D (scGPT headline + Geneformer robustness), or
   override to Option A (scGPT only) or Option C (both equally).
2. Confirm pinning a specific HuggingFace revision SHA for each
   model rather than tracking "latest" — required for
   reproducibility but adds re-pinning burden every few months.

# ADR-0001: Ground-truth construction and training-data leakage handling
Date: 2026-05-18
Status: Proposed — awaiting Armin sign-off

## Context
A probe of a pretrained model is only as honest as its ground truth.
Two ground-truth choices have to be made: (a) which genes count as
"PD-associated" for Task A, and (b) which held-out atlas is used for
Task C (and how we know it was not in the pretraining corpus).

## Options considered

### For PD-gene ground truth (Task A)
- **A1: OpenTargets PD association score ≥ 0.4 + manual curation prune.**
  Filters out promiscuous targets; keeps high-confidence calls.
- **A2: OMIM Mendelian PD genes only.** Cleanest, but very small (~7).
- **A3: GWAS + Mendelian union.** Larger, but noisier.

### For held-out atlas (Task C)
- **C1: An atlas published after the model's pretraining cutoff.**
  Cleanest claim of non-contamination.
- **C2: An atlas explicitly listed in the model paper as held out.**
  Cleanest possible.
- **C3: A random publicly available atlas.** Risk of contamination.

## Criteria
- Defensibility under hostile questioning.
- Sample size large enough for the bootstrap CI to be meaningful.
- Ability to write a one-paragraph defense in the README.

## Decision
- **A1** for the PD-gene ground truth, with an explicit second-pass
  manual curation step documented in `data/curation_pd_genes.md`.
- **C1** for the held-out atlas, with the model's pretraining cutoff
  documented in `data/README.md` and the atlas's publication date
  verified to be after that cutoff.

## Consequences
- Curation introduces a small subjective surface; the curation file is
  tracked in git so any reviewer can see exactly what was included.
- The README will note: "if the curation is wrong, the benchmark is wrong."
  This is the honest framing.

## Open questions for Armin

- Confirm C1, and identify the specific held-out atlas.
